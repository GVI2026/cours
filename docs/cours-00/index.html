Prérequis techniques

Cette page prépare l'environnement nécessaire avant les premières manipulations Git, GitHub, Docker et Dev Containers du module.

Outils de base

L'environnement de travail doit permettre d'éditer du code, d'ouvrir un terminal et d'exécuter les commandes utilisées pendant les TP.

VS Code doit être installé et capable d'ouvrir un dossier complet de projet. L'extension Git intégrée suffit pour suivre les changements, mais le terminal reste indispensable.

Git doit être disponible dans le terminal. La commande suivante doit afficher une version sans erreur : 

git --version

Git et GitHub

Un compte GitHub accessible est nécessaire pour cloner, forker, pousser des branches et ouvrir des Pull Requests pendant les exercices.

Identité Git

Le nom et l'adresse e-mail configurés dans Git apparaissent dans les commits. Les commandes suivantes permettent de vérifier la configuration globale : 

git config --global user.name
git config --global user.email

Choix SSH ou HTTPS

Deux modes d'accès sont possibles. SSH est pratique pour les TP répétés, car la clé installée évite de saisir un jeton à chaque push. HTTPS reste utilisable si l'accès SSH n'est pas prêt, avec un jeton GitHub lorsque Git le demande. 

ssh -T git@github.com

Si ce test échoue, la fiche Je suis bloqué avec GitHub donne les diagnostics rapides.

Docker et Dev Containers

Docker fournit des environnements reproductibles pour les API et les bases de données utilisées dans les TP.

Docker Desktop doit être installé et lancé avant l'ouverture d'un Dev Container. Sur Linux, Docker Engine ou Docker rootless peut remplacer Docker Desktop si les commandes Docker fonctionnent depuis le terminal.

L'extension Dev Containers de VS Code doit être installée pour rouvrir un projet dans un conteneur de développement.

Le test Docker minimal attendu est :

docker run hello-world

Le test Dev Container minimal consiste à ouvrir un projet doté d'un dossier .devcontainer/, puis à choisir Dev Containers: Reopen in Container dans VS Code. L'ouverture doit se terminer sans erreur de build ou de connexion Docker.

Points système

Windows : WSL2 doit être activé, car Docker Desktop s'appuie dessus pour fournir un environnement Linux compatible avec les conteneurs.

Linux : le compte utilisateur doit pouvoir exécuter Docker. Deux situations conviennent : appartenance au groupe docker ou installation Docker rootless opérationnelle. 

docker info

Ports à connaître

Plusieurs services des TP utilisent des ports fixes. Un port déjà occupé provoque souvent une erreur au démarrage d'un conteneur ou d'une API.

Port Usage courant
3000 API NestJS ou serveur web de développement.
3001 Exercice Docker minimal.
4873 Registry npm local Verdaccio.
2222 Serveur SSH de cible de déploiement.
5432 PostgreSQL.

Checklist finale

  • VS Code est installé et le terminal intégré fonctionne.
  • git --version affiche une version de Git.
  • Le compte GitHub est accessible depuis le navigateur.
  • Le mode d'accès GitHub est choisi : SSH prêt ou HTTPS avec jeton.
  • Docker est lancé et docker run hello-world réussit.
  • L'extension Dev Containers est installée dans VS Code.
  • Les prérequis Windows ou Linux sont vérifiés selon la machine.
  • Les ports 3000, 3001, 4873, 2222 et 5432 sont identifiés.
